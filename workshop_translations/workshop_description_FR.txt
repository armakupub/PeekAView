[h1]Élargissez votre vision[/h1]

Project Zomboid cache beaucoup de choses que votre personnage verrait réellement, derrière les murs, les arbres et les étages supérieurs. PeekAView étend trois passes de rendu, pour que votre écran se rapproche de ce que voit votre personnage. Trois fonctions indépendantes :

[list]
[*][b]Coupe de murs[/b] : les murs disparaissent à plus grande distance, pour que les coins et les portes révèlent plus tôt ce qui se trouve derrière eux quand vous approchez.
[*][b]Estompage des arbres[/b] : les arbres dans votre champ de vision deviennent translucides. Repérez les zombies derrière les rangées d'arbres là où la ligne de vue le permet ; conduite plus sûre avec une visibilité anticipée des obstacles.
[*][b]Vue d'escalier[/b] : l'étage supérieur est rendu pendant que vous montez les escaliers, pour que vous voyiez ce qui vous attend en haut avant d'y arriver.
[/list]

Mêmes règles que vanilla. Pas de rayons X, pas de murs transparents, pas de contours d'ennemis. La ligne de vue de votre personnage décide toujours de ce qui est réellement visible.

[h1]Paramètres[/h1]
Chaque fonction est indépendante. Activez les trois, juste celle dont vous avez besoin, ou un mélange. Chaque fonction a son propre interrupteur d'activation, et les portées de coupe de murs et d'estompage des arbres sont réglables dans le menu Options.

Options → Mods → Peek a View, quatre sections : Général, Coupe de murs, Estompage des arbres, Vue d'escalier. [b]F8[/b] active/désactive l'interrupteur principal en jeu (halo vert/rouge pour confirmation ; réassignable dans Options → Touches sous [noparse][PeekAView][/noparse]).

[h1]FAQ[/h1]

[b]Y a-t-il un conflit avec le mod Staircast du Workshop ?[/b]
Non. Si les deux sont abonnés, PeekAView détecte Staircast au runtime et cède sa propre vue d'escalier, donc les deux peuvent fonctionner ensemble.

[b]Fonctionne-t-il en multijoueur ?[/b]
Oui, côté client uniquement. Chaque client doit également avoir ZombieBuddy installé.

[b]J'ai désactivé PeekAView de la liste des mods en cours de session, mais les effets sont encore visibles. Que faire ?[/b]
Project Zomboid garde le code des mods en mémoire entre les rechargements de sauvegardes, donc désactiver PeekAView de la liste des mods en cours de session ne l'arrête pas tout seul. Trois façons de le supprimer : (1) le désactiver dans les options propres au mod (effet immédiat), (2) recharger votre sauvegarde (PeekAView vérifie la liste de mods active à chaque chargement et se désactive automatiquement s'il n'y est plus), ou (3) redémarrer PZ. C'est une limitation du framework déjà signalée.

[b]Cela affecte-t-il les performances ?[/b]
Plusieurs caches maintiennent le coût d'exécution proche du vanilla. Rester immobile et marcher à pied sont les chemins les moins coûteux. Réduisez les curseurs de portée si votre matériel a du mal.
