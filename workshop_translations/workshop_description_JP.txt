[h1]視界を広げる[/h1]

Project Zomboid は、キャラクターが本来見えるはずの多くを、壁、木、上の階の陰に隠しています。PeekAView は3つのレンダリングパスを拡張し、画面がキャラクターの目に近づくようにします。3つの独立した機能：

[list]
[*][b]壁カットアウェイ[/b]：壁がより遠い距離で削り取られ、近づくときに角や戸口がその先にあるものをより早く明らかにします。
[*][b]樹木のフェード[/b]：視界内の木が半透明になります。視線が届くところで木立の向こうのゾンビを見つけてください；障害物の早期視認でより安全な運転が可能になります。
[*][b]階段ビュー[/b]：階段を上っている間に上の階がレンダリングされるので、到達する前に上で待っているものが見えます。
[/list]

バニラと同じルール。X 線視はなし、透視壁はなし、敵のシルエット表示もなし。キャラクターの視線が引き続き、実際に何が見えるかを決定します。

[h1]設定[/h1]
各機能は独立しています。3つすべてを一緒に、必要な1つだけ、またはその間の組み合わせを実行できます。各機能には独自の有効化トグルがあり、壁と木のフェードの範囲はオプションメニューで調整できます。

オプション → Mod → Peek a View、4つのセクション：全般、壁カットアウェイ、樹木のフェード、階段ビュー。[b]F8[/b] はゲーム内のメインスイッチを切り替えます（緑/赤のハロで確認；オプション → キー設定の [noparse][PeekAView][/noparse] で再割り当て可能）。

[h1]FAQ[/h1]

[b]Staircast Workshop Mod と競合しますか？[/b]
いいえ。両方ともサブスクライブされている場合、PeekAView は実行時に Staircast を検出し、自身の階段ビューを譲るので、両方が一緒に動作できます。

[b]マルチプレイヤーで動作しますか？[/b]
はい、クライアント側のみです。各クライアントにも ZombieBuddy のインストールが必要です。

[b]セッション中に PeekAView を Mod リストから無効化しましたが、効果がまだ表示されます。どうすればよいですか？[/b]
Project Zomboid はセーブの再読み込みをまたいで Mod コードをメモリに保持するため、セッション中に PeekAView を Mod リストから無効化しても、Mod はそれだけでは停止しません。クリアする3つの方法：（1）Mod 自身のオプションでオフに切り替える（即時に有効）、（2）セーブを再読み込みする（PeekAView は各セーブ読み込みでアクティブ Mod リストをチェックし、もうそこにない場合は自己無効化します）、または（3）PZ を再起動する。すでに報告されているフレームワーク制限です。

[b]パフォーマンスに影響しますか？[/b]
複数のキャッシュにより、実行時のコストはバニラに近く保たれます。じっとしていることと徒歩で歩くことが最も低コストの経路です。ハードウェアが苦しい場合は範囲スライダーを下げてください。
