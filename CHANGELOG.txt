Peek a View - Changelog
=======================

1.4.2 - 2026-05-10
  - Stair view: smoother climb on player-built stairs that attach to a building's exterior wall. No more brief flash of the room behind the wall during the top-of-climb transition, and decks near the stair stay stable while climbing.
  - B42 fix: railings on player-built upper-floor decks are cut together with the floor tile, no longer drawn over the character standing below. Thanks to samuel.carlin037 on Steam for the report.
  - Default tree-fade range lowered from 20 to 15. Existing users keep their saved value.

1.4.0 - 2026-05-09
  - New "Stair view" section: while climbing stairs, the upper floor renders so you can see what's waiting upstairs. Zombies on the upper floor in your forward view become visible through the climb. If Staircast is also subscribed, PeekAView yields to it.
  - Wall cutaway: own Enable toggle, separate from the master. The "Active only when aiming" tickbox moved into the same section.
  - Tree fade: rework. The fade tracks your vision direction more accurately, and trees you've driven past refade quickly behind a moving vehicle.
  - Self-deactivates on the next save reload after removal from the mod list.

1.3.0 - 2026-05-05
  - Tree fade: smooth now, no more trees cut into pieces while fading. Trees behind the character stay visible.
  - Wall cutaway: slider values above 5 used to jump unevenly, they now step smoothly. Default range goes from 15 to 10 (one-time reset on first launch).
  - Settings tidied up, a few no-longer-needed options removed.

1.2.3 - 2026-05-03
  - Bugfix: when the player is inside a building, player-built tiles next to the building no longer bleed through cut walls. The B42 adjacency fix is now applied outdoors only, where it's needed, and indoors falls back to vanilla cutaway behavior.
  - Thanks to samuel.carlin037 on Steam for the report and the screenshots that pinned this down.
  - Removed the JAR signature requirement that v1.2.2 introduced. For users on connections that couldn't reach Steam over a clean TLS path, ZombieBuddy refused to enable the mod with a "could not verify signature" error. The signed-JAR scheme had no practical upside in return, so it's gone.

1.2.2 - 2026-05-02
  - Mod is now a signed JAR for ZombieBuddy 2.1.0+.

1.2.1 - 2026-05-01
  - Bugfix: shelves and similar objects that show the character behind them could flicker (most visible indoors with the character standing behind a shelf).
  - Thanks to laoda on Steam for the bug report and the two videos that pinned down the spot.

1.2.0 - 2026-04-28
  - Tree fade — behavior fixes:
    - Trees lined up east or south of the player along a straight axis now fade. The vanilla engine only fades trees inside a small ~5-tile area, which excluded the trees right next to a straight road — exactly the ones you drive past. They fade now, at the full slider range.
    - Tall pine trees: trunk and crown fade together. Previously the crown stayed opaque because the per-tile stencil mask didn't reach high enough on screen.
    - All four screen directions fade, not just the south-east half. A tall tree above-left of the screen-player no longer blocks the view when scanning northward. Line-of-sight still suppresses fade on trees the character can't actually see, so unseen forest stays opaque.
    - Driving symmetry fix: previously trees on one side of a treeline could snap while the other side crawled (engine and mod handle them on different code paths). Both sides now respond to driving speed identically.
  - Tree fade — speed-proportional snap: at driving speeds at or above ~30 km/h, translucent trees snap fully transparent in one frame instead of crawling at the vanilla fade rate. Below that they still ease in smoothly. No more driving into opaque crowns at highway speeds.
  - Wall cutaway slider at 5 is now pure vanilla — the patch falls through entirely. Earlier the lower bound emitted a slightly wider raster than vanilla; now "5 = vanilla" is exact.
  - Default change (existing users keep their saved values): Tree-fade driving-speed cutoff 50 → 100 km/h. With the new snap, fade keeps up at all normal driving speeds, so the old 50 km/h limit became unnecessarily restrictive. Wall-cutaway range stays at the 1.1.x default of 15.
  - Two new override tickboxes in the Tree fade section, both kick in only when "Active only in nimble stance" is enabled:
    - "Stay on while driving" (default off): keeps tree fade running in vehicles regardless of aim state.
    - "Stay on while on foot" (default on): same idea but for foot. Default on — most users enabling nimble-stance do it for the around-the-corner wall-cutaway peek, not because they want tree-fade gated; this default keeps tree fade everywhere while still letting wall cutaway respect the aim gate.
    Each override applies independently per context. The "Active up to" speed limit still applies in vehicles.
  - Performance: tile-filter on tree-fade's obscuring-tile cache. Empty grass/road tiles are dropped from the per-frame walk vanilla does over the cached list — JFR-measured CPU footprint at default settings drops by roughly half (~5% → ~2.8% of CPU samples in 120 s of driving with ~50 background mods active). The "Enable" toggle off path stays the cheapest at ~0.1%.
  - Portability fixes for B42 builds older than 42.15.0 (reported by user Mercuriae running 42.14.1, GitHub issue #1):
    - Versioned 42.13/mod.info now declares `require=\ZombieBuddy` (with leading slash) to match PZ's internal mod-id form on builds that do a literal string compare instead of normalizing.
    - Empty `common/` folder added next to `42.13/` for builds that require the multi-version layout to be present even when only one version subfolder has content.
    - Translations now ship in both formats: UI.json (42.15+) and the generated UI_<LANG>.txt Lua-table format (pre-42.15). Each PZ version picks up the format it knows; the other is ignored.

1.1.0 - 2026-04-26
  - New option "Fade trees the character can see past" (on by default): trees in your view become translucent so zombies hiding behind them are visible. Not X-ray — the engine's normal line of sight still decides what the character can actually see.
  - Mod options reorganized into three sections — Global, Wall cutaway, Tree fade — each with its own range and driving-speed settings. Wall cutaway and tree fade can now be active independently while driving (e.g. tree fade on at higher speed for spotting roadside obstacles, wall cutaway off for less screen noise).
  - Defensive handling when ZombieBuddy's JAR-loading prompt is declined for PeekAView: the options screen shows a clear warning and the F8 keybind shows a halo instead of failing silently.
  - Tooltips throughout the options screen rewritten for clarity (one sentence each, no implementation detail).
  - Performance: tree fade ships with several internal caches (per-frame gate, tree-position list) that keep its runtime cost close to vanilla — measured GC overhead within ~25 ms over 120 s of driving at maximum range.

1.0.0 - 2026-04-22
  - First stable release.
  - Default cutaway range lowered from 20 to 15 tiles.
  - New option "Active only in nimble stance" (default off): when enabled, PeekAView only kicks in while you are in nimble stance (holding right-click to aim a weapon), and turns off as soon as you lower it. (implemented on user request)
  - Note: all non-English translations (both in-game UI and Workshop description) are largely AI-assisted and may contain errors. Corrections are welcome via GitHub issues or a Workshop discussion thread.
